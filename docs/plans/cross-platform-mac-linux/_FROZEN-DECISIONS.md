# Cross-Platform Port — Frozen Decisions (canonical AD list)

> This file is the single source of truth for the Architecture Decisions and wave
> structure. Every plan-writer agent MUST treat AD numbering and wave structure as
> frozen and quote it verbatim. Generated by the planning orchestrator after the
> Phase-1 question round was answered by the maintainer on 2026-05-29.

## North Star

Personal Jarvis runs with its **full desktop power-user feature set on Windows,
macOS, and Linux** — verified by a CI matrix plus one-time live sign-offs — even
though the maintainer owns only a Windows machine. Cloud-first doctrine
(`docs/PHILOSOPHY.md`) makes macOS/Linux the primary target, not an afterthought.

## Operator decisions (locked Phase 1)

- **Order:** macOS and Linux **in parallel** — every feature wave ships both at once.
- **Ambition:** **port all 6 features**, including the Orb overlay and Admin/elevation (the hard ones).
- **Verification bar:** **CI-green + one-time live sign-off** for the GUI/permission features; per-feature honesty labels (`CI-verified` vs `live-verified <date>`).
- **CI first:** the GitHub Actions test matrix is **Wave 0, a hard prerequisite** before any port code.

## Architecture Decisions (FROZEN — AD-1 .. AD-15)

- **AD-1 — Dual-platform parallel.** Each feature wave delivers macOS and Linux simultaneously, not sequentially.
- **AD-2 — Port all six.** Terminal, App-launch, Hotkey, UI-element-click, Orb-overlay, Admin/elevation — all in scope. No descoping.
- **AD-3 — Verification = CI + one-time live sign-off.** Headless CI proves logic + real-PTY; GUI/permission behavior (Orb transparency, global hotkey capture, AX/AT-SPI tree, elevation prompt) gets one manual sign-off on a real/borrowed/rented device per feature. Docs carry an honest per-feature badge.
- **AD-4 — CI matrix is Wave 0 (blocking).** A GitHub Actions matrix on `ubuntu-latest` + `macos-latest` + `windows-latest` running `pip install -e ".[dev]"`, `ruff`, `mypy`, `pytest -m "not skip_ci"`, an **import-cleanliness gate** (`python -c "import jarvis"` on Linux/macOS proving no module-scope `pywin32`/`winreg`/`global_hotkeys` import), and a **minimum-passed-count floor** (so a mass-skip regression can't pass as green). There is **no functional CI today** (only supply-chain workflows) — this is the highest-leverage action in the whole plan. Copy the 3-OS matrix shape from `.github/workflows/cross-runner-hash.yml:76-86`.
- **AD-5 — One shared `jarvis/platform/` capability module.** `detect_platform() -> "win32"|"darwin"|"linux"` + a cached, frozen `Capabilities` dataclass (`has_hotkey`, `has_ax_tree`, `has_overlay`, `has_pty`, `has_elevation`, `display_present`, `is_wayland`, `ax_permission_granted`, …). All six ports read capabilities from here; they do NOT each re-detect. The setup wizard renders one "what works on your box" snapshot from it.
- **AD-6 — Uniform seam pattern.** Every port = `Protocol` (in or alongside `jarvis/core/protocols.py`) + per-OS implementation + `sys.platform` factory + **graceful null-fallback**. No `sys.platform` branch ever raises; an unavailable capability logs a clear English message and degrades (AD-OE6 "zero silent drops").
- **AD-7 — Permanent additive coexistence.** Windows implementations stay **untouched** ("grandfathered until touched"; BUG-009/012/014/030 fixes live in them). macOS/Linux are added as siblings behind the seam. No rewrite of the Windows path to a "unified" impl.
- **AD-8 — Hotkey: dual backend.** Windows keeps the battle-tested `global-hotkeys` (the left/right-Alt + refcount logic that BUG fixes depend on). macOS/Linux use `pynput`. Wayland: global hotkey is unavailable by OS design → no-op + log, lean on wake-word. macOS: detect "registered but zero events" → guide the user to grant Input-Monitoring/Accessibility permission.
- **AD-9 — Terminal: `ptyprocess` (Unix).** `ptyprocess.PtyProcess` mirrors all five `pywinpty` methods 1:1 (spawn/read/write/setwinsize/terminate/isalive/exitstatus) and drops into the existing daemon-thread read-loop with no async rewrite. Normalize str↔bytes at the backend seam. `shells.py` gets a Unix branch (`$SHELL` → `/etc/shells` → `which bash/zsh/fish`). Terminal is **fully real-PTY CI-testable** on ubuntu/macos runners — the standout verification win.
- **AD-10 — UI-element-click: AX + AT-SPI.** macOS via `pyobjc` (`AXUIElement` tree → `Observation`/`UIANode`); Linux via `pyatspi` (AT-SPI bus). Both satisfy the existing `VisionSource` Protocol (`protocols.py:419`) and plug into the existing DI seam at the 6 consumer sites. Native AX/AT-SPI roles are **normalized into the canonical UIA role vocabulary** (`pruning.py:51`, `_CLICKABLE_UIA_ROLES`) so the model prompt + tests stay platform-agnostic. Degrades to the already-working pixel-click path when the native tree is empty/unavailable.
- **AD-11 — Orb: `OverlaySurface` abstraction + 3-tier visual ladder.** `TkColorKeyOverlay` (Windows + macOS — `-transparentcolor` works on both) / best-effort transparent surface on Linux+compositor / `TrayOnlySurface` fallback driving the **already-cross-platform pystray tray** (`jarvis/ui/tray.py`, no platform marker). The Windows system-cursor swap (`SetSystemCursor`) stays Windows-only (no OS equivalent; already a no-op off-Windows). NOTE: a framework conflict surfaced in research (Tk in `jarvis/ui/orb/` vs PySide6 in `OS-Level/src/overlay/`) — Wave 0 must establish which is the live orb before this wave starts.
- **AD-12 — Admin/elevation: full port behind `Elevator` + `AdminTransport` seams.** The transport-free HMAC/envelope/Pydantic-argv layer (`ipc.py:65-262`) is reused as-is (it is the security core; preserve no-`shell=True`, pattern-validated argv). New per-OS op vocabulary (macOS `brew`/`launchctl`/protected-path via Authorization Services or `osascript … with administrator privileges`; Linux `apt`/`systemctl`/`ufw`/protected-path via `pkexec`/polkit or `sudo`). `UnixSocketTransport` (0700 socket in `$XDG_RUNTIME_DIR`, `SO_PEERCRED`/`LOCAL_PEERCRED` peer-cred check) replaces the Windows SDDL-ACL named pipe. `NullElevator` is the headless/no-auth fallback (refuse with a clear message). This is the largest, most security-sensitive wave; never CI-testable end-to-end (interactive auth) → relies on AD-3 live sign-off + heavy unit tests with a fake transport.
- **AD-13 — Permission-UX is detect-and-degrade.** macOS AX/Input-Monitoring and Linux AT-SPI-bus: probe at first use; if missing, log an English onboarding message + fall back, never silently empty (recreates the "said it worked, nothing happened" class) and never hard-block (breaks the always-degrade contract).
- **AD-14 — Dependency grouping.** `pynput` + `ptyprocess` → `[desktop]` extras, **no platform marker** (all-platform; `ptyprocess` marked `sys_platform != 'win32'`). `pyobjc-framework-{Quartz,ApplicationServices,Accessibility}` → new `[desktop-macos]` extra, `sys_platform == 'darwin'`. Linux `pyatspi` is **NOT on PyPI** (GObject-Introspection, distro-packaged: `apt install python3-pyatspi gir1.2-atspi-2.0`) → documented as a system prerequisite + a `capabilities.has_ax_tree` runtime probe, NOT a pip extra. Mirror the existing `pyproject.toml:99-110` `sys_platform=='win32'` pattern.
- **AD-15 — `KNOWN_APPS` whitelist is platform-conditional.** Selected by `sys.platform` (Windows names today; `Safari`/`Terminal`/`firefox`/`nautilus`/`gnome-calculator` etc. per OS), keeping the anti-STT-hallucination gate intact with correct per-OS names + the PATH/URL/path escape hatches.

## Wave structure (FROZEN)

- **Wave 0 — Foundation (BLOCKING):** shared `jarvis/platform/` (detector + `Capabilities` + probes); GitHub Actions Win+Mac+Linux pytest/ruff/mypy matrix + import-cleanliness gate + min-passed-floor; resolve the Orb-framework conflict (PC-6); clean the stale `ui.orb` conftest path. Nothing else may merge until the matrix is green.
- **Wave 1 — Easy / CI-provable (3 parallel worktrees):** Terminal (Unix-PTY, ptyprocess + Unix shell discovery), App-launch (`open`/`xdg-open` + platform `KNOWN_APPS`), Hotkey (pynput dual-backend). All three independent.
- **Wave 2 — Permission/GUI-heavy (parallel):** UI-element-click (macOS AX + Linux AT-SPI behind `VisionSource`), Orb (`OverlaySurface` ladder: Mac transparent, Linux best-effort + tray fallback).
- **Wave 3 — Admin/elevation (security-sensitive):** `Elevator` + `AdminTransport` seams; per-OS op vocabulary; `UnixSocketTransport` + peer-cred; supersede ADR-0001 with a new ADR-0020; `NullElevator` headless fallback.
- **Wave 4 — Hardening + sign-off:** the one-time live sign-off round on real macOS + Linux for the AD-3 GUI/permission features; honest per-feature verification labels in docs; run the JARVIS-20 cross-platform benchmark.

## Stakeholder-Conflict Matrix (resolved)

| Axis | Decision | Why |
|---|---|---|
| Windows-tempo vs cross-platform | **Cross-platform, additively** (AD-7) | Doctrine makes Mac/Linux primary; keep Windows untouched to avoid re-opening closed bugs |
| Code-cleanliness vs fallback-safety | **Permanent coexistence** (AD-7) | Battle-tested Windows impls stay; unify only the seam, not the impls |
| Verification rigor vs no-hardware reality | **CI + one-time live sign-off + honest labels** (AD-3) | Full real-hardware E2E is impossible for a Windows-only maintainer; CI-only ships untested GUI paths |
| Feature-completeness vs risk | **Port all, sequenced easy→hard** (AD-2 + waves) | Operator chose full scope; sequencing puts the CI-provable wins first and the un-CI-able Admin last |
| Security vs convenience (Admin) | **Preserve the HMAC/validated-argv core; NullElevator default** (AD-12) | Elevation is the highest-risk surface; never weaken the injection defenses to add convenience |

## Definition of Done (EK-1 .. EK-6)

- **EK-1:** CI matrix green on Windows + macOS + Linux for `ruff` + `mypy` + `pytest -m "not skip_ci"`, with the import-cleanliness gate and min-passed-floor enforced.
- **EK-2:** Each of the 6 features has a per-OS implementation behind its seam, selected by the shared `jarvis/platform/` factory, degrading to a logged no-op when the capability is absent.
- **EK-3:** Every new OS seam ships a `tests/fakes/` fake and unit tests; no `unittest.mock`.
- **EK-4:** Terminal + App-launch resolution + Hotkey logic are CI-verified on the ubuntu+macos runners (terminal via a real PTY).
- **EK-5:** The GUI/permission features (UI-element-click, Orb, Hotkey live capture) each carry a one-time live sign-off note (`live-verified <date> on <device>`) or an explicit `unverified-on-real-desktop` label.
- **EK-6:** ADR-0001 superseded by a new ADR; CLAUDE.md + README capability matrix updated to reflect real cross-platform status; no module hard-imports a Windows-only package at module scope.
