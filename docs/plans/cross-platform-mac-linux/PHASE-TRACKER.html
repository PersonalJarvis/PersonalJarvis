Recovery patterns

      
If something goes wrong, find the matching pattern below and paste its prompt into a fresh Claude Code session. You never debug by hand — each prompt snapshots state before any destructive action and never force-pushes or bypasses the CI gate.

    

  


  


    
    

      

        

          
Recovery

          
Resume a rate-limited / interrupted agent

          
An agent stalls or says it is rate-limited / out of quota mid-task. Re-points a fresh session at the same worktree and continues from the last green acceptance command.

        

      

      

        

          PROMPT recovery-rate-limited
          

            
            
          

        

        

      

    


    
    

      

        

          
Recovery

          
Red CI — diagnose and fix the failing leg

          
The matrix goes red (ruff, mypy, a test, the import gate, or the min-passed floor). Reads the failing GitHub Actions log, identifies the OS leg + gate, fixes it, and re-watches the matrix.

        

      

      

        

          PROMPT recovery-red-ci
          

            
            
          

        

        

      

    


    
    

      

        

          
Recovery

          
Resolve a merge conflict between two wave worktrees

          
Two worktrees touched the same file (almost always pyproject.toml in Wave 1's 1.5 or Wave 2's 2.7). Takes both additive edits and re-runs the acceptance set before merging.

        

      

      

        

          PROMPT recovery-merge-conflict
          

            
            
          

        

        

      

    


    
    

      

        

          
Recovery

          
Unstick a worktree (preflight + editable-install check)

          
A worktree is stuck / the agent can't find files / "works in tests but Jarvis behavior unchanged." Runs scripts/preflight.ps1 and confirms import jarvis points at the worktree clone (the four-layer restore trap, BUG-006/014).

        

      

      

        

          PROMPT recovery-worktree-stuck
