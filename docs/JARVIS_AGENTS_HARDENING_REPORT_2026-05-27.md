# Jarvis-Agent Spawning Pipeline — Hardening Report

## Executive summary

The Jarvis-Agent spawning pipeline is **production-ready with caveats** — there are no critical/silent-data-loss-of-everything or crash-class defects, and the AD-OE6 zero-silent-drops invariant holds (every failure path still produces a spoken announcement). The sharpest residual risk is **deliverable artifact loss for a definable subset of filenames**: a German/bilingual assistant routinely produces umlaut filenames (`Werbungä.html`, `Lebenslauf-Müller.pdf`), and these are silently dropped from `artifacts/files/` because the archive path trusts git's `core.quotepath` octal-escaped output (`archive-newfile-octal-escape-drops-nonascii-deliverable`, HIGH); the same archive path also drops any `.gitignore`-matched deliverable name (`*.log`, `dist/*`, `*.db`) since it relies 100% on git path enumeration with no filesystem fallback (`archive-untracked-copy-relies-on-git-enumeration-not-filesystem`, MEDIUM). The next tier is **two false-failure / false-block UX defects in the spawn + critic loop**: a fast mission failure leaves the 30s spawn cooldown armed so a legitimate retry gets a false "already running" ACK (`spawn-cooldown-no-failure-reset`, MEDIUM), and edit-only tasks whose final diff is byte-identical to HEAD burn all 3 critic loops into `critic_loop_exhausted` despite valid work (`critic-empty-diff-pregate-revise-burns-all-loops-on-edit-only-tasks`, MEDIUM). The remaining findings are low-severity (a concurrent-spawn race with a one-extra-mission worst case, a latent double-readback footgun, an unreachable crash-recovery template, and diagnostic opacity on worktree-create failure). None block shipping, but the two HIGH/MEDIUM archive findings should be fixed before the umlaut-deliverable failure recurs in front of the user.  <!-- i18n-allow -->

## HIGH

### Non-ASCII deliverable filenames are silently dropped from artifacts/files/ (git core.quotepath octal-escape not decoded)

**What breaks.** When a worker creates a deliverable with a non-ASCII name (umlauts: `Werbungä.html`, `Lebenslauf-Müller.pdf`), the archive step fails to copy it into `artifacts/files/`. Mission ends APPROVED, `diff.patch` carries the content, but the Outputs UI shows no deliverable file and `build_deliverable_summary` returns `''`, so voice falls back to the generic "Mission completed". For a bilingual DE assistant, umlaut filenames are routine, so this recurs. <!-- i18n-allow -->

**Evidence (`jarvis/missions/kontrollierer/orchestrator.py`).**
- Lines 179-182: `_NEW_FILE_DIFF_HEADER_RE = re.compile(r'^diff --git (\"?)a/(?P<path>[^\"\n]+?)\1 (\"?)b/(?P=path)\3\nnew file mode', re.MULTILINE)`.
- Lines 202-206: `_extract_new_file_paths_from_diff` only strips surrounding quotes (`if raw.startswith('"') and raw.endswith('"'): raw = raw[1:-1]`) — it never decodes git's C-style octal escapes.
- Lines 1209-1219: the recovered paths feed `src = worktree / rel; if not src.is_file(): continue`.
- With default `core.quotepath=true`, `git diff --cached HEAD` emits `diff --git "a/Werbung\303\244.html" "b/Werbung\303\244.html"`. The regex returns the literal `Werbung\303\244.html`, so `worktree / 'Werbung\303\244.html'`.is_file() is `False` while the real file `Werbungä.html` exists. The `git ls-files --others` fallback (line 1148) is also empty because `_capture_diff` already ran `git add -A` (line 1009-1010). No `core.quotepath=false` anywhere in `jarvis/` (grep = 0 hits); the dedicated test `tests/missions/kontrollierer/test_extract_new_file_paths.py` covers ASCII names only.  <!-- i18n-allow -->

**Repro.** Voice: "baue eine Werbeseite und nenne sie Werbungä.html". Mission ends APPROVED; `artifacts/files/` is empty; Outputs UI shows no deliverable; voice says generic "Mission abgeschlossen."  <!-- i18n-allow -->

**Proposed fix (code-level).** Prefer (a) — it matches the existing `git -c core.longpaths=true` injection pattern in `worktree.py:347`. Add `'-c', 'core.quotepath=false'` to the `git diff --cached HEAD` invocation in `_capture_diff` AND to the `git ls-files --others --exclude-standard` invocation in `_archive_task_artifacts`, so git emits raw UTF-8 paths:

```python
# _capture_diff
subprocess.run(["git", "-c", "core.quotepath=false", "diff", "--cached", "HEAD"], ...)
# _archive_task_artifacts
subprocess.run(["git", "-c", "core.quotepath=false", "ls-files", "--others", "--exclude-standard"], ...)
```

Add a regression fixture with a non-ASCII filename (`Werbungä.html`) to `tests/missions/kontrollierer/test_extract_new_file_paths.py` and an end-to-end archive test. (b) — decoding the octal escapes in `_extract_new_file_paths_from_diff` — is the fallback if any call site can't take the `-c` flag.  <!-- i18n-allow -->

## MEDIUM / LOW (condensed)

- **`spawn-cooldown-no-failure-reset` (medium, duplicate-spawn) — `spawn_openclaw.py:281-298,369`.** `_last_spawn_at` is armed at line 369 and never reset on any terminal state; a fast mission failure (decompose_failed, ~2-3s) is spoken as failed via `listener.py:124-125`, but the 30s cooldown stays armed, so a distinct follow-up heavy-work request inside the window gets the false "Already working on it, running in background." ACK and is **not** dispatched.  <!-- i18n-allow --> **Fix:** treat cooldown as a liveness gate — add `_active_dispatches: int`, increment before `create_task`, decrement in a `finally:` inside `_background_dispatch`; suppress only when `_active_dispatches > 0 AND within window`. (Or reset `_last_spawn_at = 0.0` on every terminal state.) **Needs a reset-on-failure regression test** (none exists). Bounded/self-healing at 30s → medium not high.
- **`critic-empty-diff-pregate-revise-burns-all-loops-on-edit-only-tasks` (medium, other) — `critic/runner.py:499-547`.** Write-shaped tool_use sets `_is_readonly_task=False` (491-497); an empty diff that is empty *because the requested state already holds* (Edit re-applies content byte-identical to HEAD on a tracked file → `git diff --cached HEAD` empty) hits the deterministic `revise`, confidence 1.0, no LLM (538-547), 3x → `critic_loop_exhausted`. **Fix:** when a write-shaped tool_use IS present but the diff is empty, treat it like the read-only path (let the Critic LLM grade on-disk state) rather than auto-revise; or assert the post-condition. **Needs a fixture** for edit-to-identical-content in `tests/missions/critic`. Narrow trigger (tracked file, byte-identical) → medium.
- **`archive-untracked-copy-relies-on-git-enumeration-not-filesystem` (medium, missing-artifact) — `orchestrator.py:1143-1228`.** Deliverable copy is 100% git-enumeration (`ls-files --others --exclude-standard` ∪ diff-parsed paths); a deliverable whose name matches a committed `.gitignore` pattern (`/*.log`, `dist/`, `*.db`, `.env`) is dropped from BOTH sources and from `diff.patch`, then lost when the worktree is removed. **Fix:** walk the worktree filesystem (`Path.rglob`, excluding `.git/`, `openclaw_state/`, materialized contract files) and diff against the base-branch tree instead of trusting git enumeration; capture the untracked set inside `_capture_diff`'s FIRST invocation (before its `git add -A`). **Needs a gitignore'd-deliverable test.** Same-file as the HIGH finding — fix in one commit.
- **`cooldown-arm-after-await-concurrent-fragment-race` (low, race) — `spawn_openclaw.py:281-369`.** Cooldown read (281-285) and arm (369) straddle the real suspension `await self._bus.publish(OpenClawAnnouncement(...))` (351, yields per `bus.py:57`). Two concurrent execute() coroutines (REST + voice in same loop tick) can both pass and double-spawn. VAD-fragmented turns arrive serially so the common case is safe; worst outcome one extra mission. **Fix:** move `self._last_spawn_at = now` to right after the suppress-check block (~line 298), rolling back on the early-return failure paths; or wrap check+arm in an `asyncio.Lock`. Combine with the cooldown-reset fix.
- **`double-voice-readback-listener-and-announcer-both-active` (low, voice-readback) — `missions/init.py:462-493`.** The two start-blocks are independently gated with no mutual-exclusion guard; passing both `tts_speak_fn` AND `speech_bus` produces a double announcement (announcer docstring at `announcer.py:27-29` confirms the mode). No live caller triggers it today (`server.py:1313/1318` passes `tts_speak_fn=None` + `speech_bus`), so latent footgun. **Fix:** make paths mutually exclusive in bootstrap — prefer `MissionAnnouncer` when `speech_bus` is set, skip the listener, with an explicit log line.
- **`crash-recovery-rendered-as-generic-failed-not-crash-template` (low, crash-recovery) — `voice/listener.py:124-125`.** The `MissionFailed` branch never special-cases `reason=='crash_recovery'`, so it would speak raw "Grund: crash_recovery" instead of `render_crash_recovery()`. Currently **unreachable** in shipped wiring (listener requires `tts_speak_fn`, which `server.py:1313` hardcodes `None`); the active announcer maps it correctly. **Fix:** in `listener._render`, mirror the announcer's reason→phrase map (fixes ALL raw snake_case reasons, not just crash_recovery) before the generic `render_failed` fallback. Same-file class as the cooldown/race — but different module; low priority.
- **`unsupported-gate-preempts-llm-on-universal-only-verbs` (low, missing-artifact) — `brain/manager.py:2082-2089,1374-1387`.** UNSUPPORTED gate runs before the LLM loop and hard-refuses utterances whose only verb is a universal-but-unseeded one (`bestell/kauf/buch/ruf/post/tweet/reserviere`). This is the intended ADR-0017 anti-hallucination design (asserted in `test_routing.py:880-915`); in strict force-spawn mode it does NOT pre-empt a spawn (force-spawn wasn't going to fire on those verbs anyway). Residual concern is narrow over-refusal of router-servable phrases. **Fix (optional):** prune `_UNIVERSAL_ACTION_VERBS` of order/buy/book-class verbs that have no router path, or make the gate the last resort after the LLM tool-choice attempt.
- **`worktree-create-failure-returns-error-no-spoken-feedback` (low, silent-drop) — `orchestrator.py:412-419`.** A worktree-create failure (200-char path cap `ValueError` from `isolation/worktree.py:114`, or git index-lock `CalledProcessError`) maps to the generic `task_error` readback ("Der Worker ist abgebrochen.") indistinguishable from a real worker crash; the git stderr / path-length cause survives only in `logger.exception` (418) and is lost from the spoken reason. NOT a silence bug (announcer still speaks). **Fix:** add a distinct `TaskOutcome.SETUP_FAILED` outcome or thread the exception text into `_fail_mission`'s reason so the readback can say "Konnte keinen Arbeitsbereich anlegen."  <!-- i18n-allow -->

No octal-escape / umlaut / `.gitignore`-matched-deliverable case. Add fixtures for both archive findings.

- **Spawn cooldown concurrency (race class):** the same file's docstring states it tests the suppress branch "in isolation by pre-arming"; no two-concurrent-`execute()` coroutine test exercises the check-and-arm-across-await window.
- **Critic empty-diff-with-write-tool (false-failure class):** `tests/missions/critic` has zero coverage of the readonly-path or edit-to-identical-content scenario; the only related test (`test_runner_dryrun.py:204`) asserts the OPPOSITE (empty-diff → hard veto). Add an edit-to-identical-content fixture.
- **Worktree-setup failure outcome (silent-drop / diagnostic class):** grep for `SETUP_FAILED` / worktree-create-fail in tests = 0 matches. No test exercises the path-cap `ValueError` or git-index-lock `CalledProcessError` branch.

- **Double-readback bootstrap guard (voice-readback class):** no test asserts that `bootstrap_missions(tts_speak_fn=fn, speech_bus=bus)` activates exactly one readback path; the only protection is a docstring.
